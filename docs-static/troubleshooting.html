Troubleshooting

calimero-sdk-js / troubleshooting

Build Issues

QuickJS Not Found

Error

QuickJS compiler not found

Cause

QuickJS is downloaded automatically on install but may not have run.

Fix

pnpm install # downloads QuickJS automatically

WASI-SDK Not Found

Error

WASI-SDK not found

Cause

The CLI package downloads WASI-SDK on first install.

Fix

cd packages/cli
pnpm install # downloads WASI-SDK

Unsupported Platform

Error

Platform win32 is not supported

Supported

macOS (darwin) and Linux only.

Workaround

Use WSL (Windows Subsystem for Linux) on Windows. The full Linux toolchain works inside WSL.

TypeScript / Module Errors

Error

Cannot find module '@calimero-network/calimero-sdk-js'

Fix

pnpm install
pnpm build # build SDK packages first

Runtime Issues

Method Not Found

Error

Contract method not exported at runtime.

Checklist

  • Method is inside a @Logic class
  • Method is not named constructor
  • Build succeeded — check methods.h was generated
// ✅ Correct — method in @Logic
@Logic(MyApp)
export class MyAppLogic extends MyApp {
  myMethod(): void { /* … */ } // exported
}

// ❌ Wrong — method in @State
@State
export class MyApp {
  myMethod(): void {} // not exported!
}

Storage Errors

Error

Storage operation failed

Checklist

  • Check key serialization — keys must be Uint8Array
  • Verify storage access permissions
  • Check for null/undefined values before write
// ✅ Correct — Uint8Array key
const key = new TextEncoder().encode('my-key');
const val = new TextEncoder().encode(data);
env.storageWrite(key, val);

// Guard nulls
const item = this.map.get(id);
if (!item) throw new Error('not found');

Event Handler Not Executing

Possible Causes

  • Handler name doesn't match the method name string passed to emitWithHandler
  • Author node trying to execute its own handler (this is by design — it doesn't)
  • Delta not applied — parents missing from peer's DAG
// Handler name must match exactly
emitWithHandler(event, 'onItemAdded');

// Method must have this exact name
onItemAdded(e: ItemAdded): void { /* … */ }

// Debug: log the handler name
env.log(`emitting: onItemAdded`);

Performance Issues

Slow Build Times

For development

calimero-sdk build src/index.ts --no-optimize

Skip WASM optimisation during dev — saves 10-30s per build.

For production

calimero-sdk build src/index.ts --optimize

Full wasm-opt run. Required for production deploys to minimise binary size.

Large WASM Binary

Expected size

~500 KB is normal. QuickJS runtime alone is ~450 KB.

Reduce size

  • Enable --optimize flag
  • Remove unused imports
  • Avoid importing entire libraries when only using one function
