Getting Started

calimero-sdk-js / getting-started

Prerequisites

Node.js ≥ 18

Required for running the CLI toolchain and build pipeline.

pnpm ≥ 8

Recommended package manager. npm and yarn also work.

merod running

A Calimero node must be running locally or remotely to deploy services.

Installation

Install both the SDK (used in your service code) and the CLI (build toolchain):

# npm
npm install @calimero-network/calimero-sdk-js
npm install -D @calimero-network/calimero-cli-js

# or pnpm
pnpm add @calimero-network/calimero-sdk-js
pnpm add -D @calimero-network/calimero-cli-js

Create Your First App

STEP 1

Create project structure

mkdir my-calimero-app && cd my-calimero-app
pnpm init
pnpm add @calimero-network/calimero-sdk-js
pnpm add -D @calimero-network/calimero-cli-js typescript
STEP 2

Write your contract — src/index.ts

import { State, Logic, Init, View, createCounter } from '@calimero-network/calimero-sdk-js';
import type { Counter } from '@calimero-network/calimero-sdk-js/collections';
import * as env from '@calimero-network/calimero-sdk-js/env';

@State
export class CounterApp {
  count: Counter = createCounter();
}

@Logic(CounterApp)
export class CounterLogic extends CounterApp {
  @Init
  static initialize(): CounterApp {
    env.log('Initializing counter');
    return new CounterApp();
  }

  increment(): void {
    this.count.increment();
    env.log('Counter incremented');
  }

  @View()
  getCount(): bigint {
    return this.count.value();
  }
}

Initialize CRDT fields inline

Call createCounter() in the field initializer so the runtime rehydrates persisted state without relying on constructor logic.

Mark read-only with @View()

View methods skip persistence and avoid emitting redundant storage deltas when only returning data.

STEP 3

Build your service

npx calimero-sdk build src/index.ts -o build/service.wasm

This runs the full pipeline: TypeScript → JavaScript → ABI extraction → QuickJS bytecode → Clang/WASI → service.wasm.

STEP 4

Deploy the service

meroctl --node-name node1 app install \
  --path build/service.wasm \
  --context-id <YOUR_CONTEXT_ID>
STEP 5

Call your service

# Increment the counter
meroctl --node-name node1 call \
  --context-id <CONTEXT_ID> \
  --method increment

# Get the count
meroctl --node-name node1 call \
  --context-id <CONTEXT_ID> \
  --method getCount

Private Storage

Use private storage for node-local data that should not replicate to peers — e.g. cached secrets, per-node counters:

import { createPrivateEntry } from '@calimero-network/calimero-sdk-js';

const secrets = createPrivateEntry<{ token: string }>('private:secrets');

const current = secrets.getOrInit(() => ({ token: '' }));
secrets.modify(
  value => { value.token = 'rotated-token'; },
  () => ({ token: '' })
);

Entries route through storage_read / storage_write directly — they never appear in CRDT deltas.

Common Issues

Build Errors

FIX

Cannot find dependencies

Run pnpm install — QuickJS and WASI-SDK are downloaded automatically on first install.

FIX

TypeScript errors

Run pnpm list typescript to verify version ≥ 5.3. Use --verbose flag for detailed output.

Runtime Errors

FIX

Method not found

Ensure the method is inside a @Logic decorated class and is not the constructor.

FIX

Service fails at runtime

Check logs with meroctl logs. Verify host functions are available and error handling is in place.
