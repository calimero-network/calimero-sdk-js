What is Calimero SDK JS?

The Calimero JavaScript SDK lets you write Calimero P2P services in TypeScript/JavaScript. Your code compiles to a WASM binary (via QuickJS) that runs inside a Calimero node, with automatic state replication across all peers using CRDTs.

1

Write TypeScript

Use familiar decorators (@State, @Logic, @Init) and CRDT collection types to define your application logic.

2

Build to WASM

The CLI compiles TypeScript → JavaScript → QuickJS bytecode → WASM. The ABI manifest is embedded automatically for Rust compatibility.

3

Deploy & Sync

Install on a Calimero node. State changes replicate to all peers via causal deltas — no manual sync code required.

Quick Start

Install

# SDK for your service code
npm install @calimero-network/calimero-sdk-js

# CLI build toolchain
npm install -D @calimero-network/calimero-cli-js

Write your first service

import { State, Logic, Init, createCounter } from '@calimero-network/calimero-sdk-js';
import * as env from '@calimero-network/calimero-sdk-js/env';

@State
export class CounterApp {
  count = createCounter();
}

@Logic(CounterApp)
export class CounterLogic extends CounterApp {
  @Init
  static initialize(): CounterApp { return new CounterApp(); }

  increment(): void { this.count.increment(); }
  @View()
  getCount(): bigint { return this.count.value(); }
}

Build & Deploy

# Build to WASM
npx calimero-sdk build src/index.ts -o build/service.wasm

# Install on node
meroctl --node-name node1 app install --path build/service.wasm --context-id <ID>
